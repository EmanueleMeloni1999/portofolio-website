Asset Preparation

From Unreal to Maya

The model was sourced directly from Unreal Engine as a Skeletal Mesh and exported as FBX. The resulting triangulated, single-shape mesh required extensive cleanup: separating components, planning the joint hierarchy, and creating quad-based proxy meshes for organic deformation.

Unreal Export

The original Skeletal Mesh exported from Unreal Engine. A single triangulated shape containing all character components combined together.

Mesh Cleanup & Hierarchy

Components separated and grouped according to the rig's requirements. This phase allowed drafting a preliminary joint list and provided a clear overview of the project scope.

Proxy Meshes

Quad-based proxy meshes created for sections requiring organic deformation, such as the spine, cables, and arms. Essential for precise skinning and smooth deformation.

Quality Control

A custom model-checking tool ensures the asset meets all technical standards: frozen transformations, zeroed pivots, removal of namespaces, and clean construction history.

Materials

Texture Baking & Preview

Since the original materials were procedural shaders built in Unreal Engine, a baking process was performed to generate standard texture maps for high-fidelity preview in Maya.

Unreal Shader Analysis

Procedural shader networks identified and prepared for baking. Each material channel was mapped to ensure accurate texture extraction from the engine.

Baked Textures

Standard texture maps generated from the procedural shaders, providing animators with an accurate visual reference during the rigging and animation workflow.

Maya Preview

Baked textures applied in Maya, giving a high-fidelity visual preview essential for accurate animation work and professional quality control.

Pipeline

Rigging Process

The rigging process followed a structured and iterative pipeline leveraging custom automation tools, mGear guide templates, and a standardized directory structure to ensure technical compliance and high-quality deformations.

01

Guide Placement

Primary mGear guide positioning and control shape customization on the target character.

02

Rig Build

Automated rig generation with Pre/Post custom scripts through the pipeline tools.

03

Primary Skinning

Base deformations established with skin weight influence strictly capped at 4 for engine compatibility.

04

Secondary Controls & Features

Weapon support system, cables, spine mechanics, and extra feature implementation.

05

Validation & Export

Animation testing, control picker creation, and Skeletal Mesh export to Unreal Engine.

Rig Features

Control Systems & Features

A comprehensive IK system, pre-configured easy poses, RBF-driven correctives, dynamic wires, full weapon integration with space switches, and a dedicated control picker.

mGear Guides

The mGear guide template built directly onto the model, serving as the blueprint for the final rig with precise control over joint placement and behavior.

Rig Overview

Full character rig with IK/FK switching on limbs and spine, space switching on weapons and head, and custom control shapes designed for intuitive animator interaction.

Control Picker

A dedicated picker interface for quick selection of all body and weapon controls. Organized for efficient animator workflow with clear visual feedback.

Skeletal Mesh

Final Skeletal Mesh exported and re-imported into Unreal Engine, maintaining full compatibility with the engine's animation system and 4-influence skin limit.
